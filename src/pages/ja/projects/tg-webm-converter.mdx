### プロジェクト概要

`tg-webm-converter` は、画像や動画ファイルをTelegramのステッカー/アイコン要件に沿ったWebM形式へ自動変換するPython製CLIツールです。

個人的にステッカー作成の手間を減らしたいという目的から生まれ、手動変換の工程を排除しました。
静止画とアニメーションを同一のステッカーパックで扱えるようにし、変換作業をスムーズにします。

PyPIで配布しているため、さまざまなOSから利用できます。

### 技術スタック

- **Python**: コアロジックとCLIインターフェース。
- **Poetry**: 依存管理とパッケージング。
- **FFmpeg**: 高性能なメディア処理と最適化。

### 主要機能

- バッチ変換
- Telegramステッカー要件への準拠:
  - 512x512のWebMステッカー（最大256KB）を生成
  - 専用の `--icon-file` 引数で100x100のWebMアイコン（最大32KB）を生成
- 自動最適化
  - FFmpegで効率的にエンコード・サイズ削減し、厳格なサイズ制限を満たす
- クロスプラットフォーム対応
  - シェルスクリプトからPython CLIに書き直し、`bash`/`zsh`以外（Windowsなど）でも動作
- ワークフローの簡素化
  - 複雑なFFmpegコマンドを置き換え、Telegramステッカー作成をプログラムで完結

### 解決する課題

Telegramユーザーとして、メディアファイルをステッカー要件に合わせて手動変換することや、
静止画とアニメーションのステッカーパックを分けて管理することに大きな負担がありました。

`tg-webm-converter` は次の点でこの課題を解消します。

1.  **煩雑な変換を自動化:** 画像編集ツールや複雑なFFmpegコマンドを不要に。
2.  **統一されたステッカーパック:** すべてをWebMに変換し、静止画とアニメーションを一つのパックで管理。
3.  **利用しやすいCLI:** `pip` で導入でき、幅広いユーザーに利用可能。

### ハイライト

Python 3.8+で開発し、精密な引数解析による対象指定やディレクトリ単位の処理に対応しています。

PyPIでの公開により `pip install tg-webm-converter` で簡単に導入可能です。

GUI対応や大規模バッチの性能最適化など、今後の拡張も視野に入れています。
